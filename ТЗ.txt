# Техническое задание: Top-Down Shooter

## 1. Цели проекта
- Создать 2D top-down shooter с несколькими уровнями, системой прогрессии и финальным экраном результатов.
- Обеспечить стабильную работу на целевой платформе, высокую отзывчивость управления и понятные игровые правила.
- Подготовить полный набор артефактов (исходники, ресурсы, инструкции запуска) для передачи заказчику.

## 2. Платформа и движок
- Платформа: ПК (Windows/Linux/macOS).
- Технологии: Python 3.x + игровой фреймворк (например, Pygame) + физический движок (например, pymunk/box2d).
- Система сборки/запуска: `python -m venv` + `pip install -r requirements.txt`.

## 3. Функциональные требования (фичи)
### 3.1 Игровые экраны
- **Стартовое окно**: кнопка «Начать», описание управления, выбор уровня (если доступно).
- **Финальное окно**: итоговая статистика, кнопка «Повторить» и «Выйти».

### 3.2 Игровой процесс
- Игрок управляет персонажем сверху (WASD/стрелки + мышь для прицеливания/стрельбы).
- Несколько уровней с возрастающей сложностью.
- Камера, следящая за игроком (плавное следование, ограничение по границам уровня).
- Система частиц (всплески, выстрелы, дым, попадания).

### 3.3 Графика и анимации
- Спрайты персонажа, врагов, окружения и снарядов.
- Анимация (ходьба, стрельба, смерть/уничтожение).

### 3.4 Физика и столкновения
- Collide-система для игрока, врагов, препятствий и снарядов.
- Физический движок для обработки столкновений и движения.

### 3.5 Звук
- Фоновая музыка.
- Звуки выстрелов, попаданий, шагов, поражений и интерфейса.

### 3.6 Подсчёт и вывод результатов
- Счётчик очков, врагов, времени прохождения.
- Вывод результатов в HUD и на финальном экране.

### 3.7 Хранение данных
- Сохранение статистики/прогресса в файл (txt/csv) или БД (SQLite) — по выбору реализации.

### 3.8 Зависимости
- Наличие файла `requirements.txt` со списком зависимостей (pygame, физический движок и пр.).

## 4. Нефункциональные требования
- FPS: не менее 60 на средней конфигурации.
- Устойчивость: отсутствие критических ошибок при типовых сценариях.
- Читаемость кода, модульная структура, комментарии к ключевым системам.

## 5. Критерии приёмки
- Реализованы стартовое и финальное окна.
- Присутствуют все обязательные фичи: подсчёт/вывод результатов, спрайты, collide, анимация, несколько уровней, камера, система частиц, звук, физический движок, хранение данных.
- В `requirements.txt` перечислены все зависимости.
- Сборка/запуск возможны по инструкции без ручной настройки.
- Игра проходит от старта до финального экрана без критических ошибок.

## 6. Артефакты
- Исходный код проекта.
- Игровые ресурсы (спрайты, звуки, шрифты).
- Инструкция запуска/установки (README или отдельный файл).
- Файл `requirements.txt`.
